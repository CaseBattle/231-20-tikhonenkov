🏠 Use Case Diagram:
Что это: Показывает, кто и что может делать в системе
Простыми словами: Это список возможностей для разных пользователей:

Арендатор: искать жилье, бронировать, платить

Арендодатель: добавлять объявления, управлять ими

Администратор: следить за порядком в системе

🗃️ Class Diagram:
Что это: Структура базы данных
Простыми словами: Это схема таблиц, где хранится информация:

Таблица пользователей (кто зарегистрирован)

Таблица жилья (какие квартиры/дома сдаются)

Таблица бронирований (кто когда снял жилье)

Таблица платежей (кто сколько заплатил)
И связи между ними - какая таблица с какой связана

🔄 Sequence Diagram:
Что это: Пошаговый сценарий работы системы
Простыми словами: Показывает, что происходит, когда пользователь делает какое-то действие:

Пользователь ищет жилье

Система проверяет базу данных

База возвращает список вариантов

Система показывает результаты пользователю
И так для каждого действия

🛣️ Activity Diagram:
Что это: Все возможные пути при работе с системой
Простыми словами: Показывает разные варианты развития событий:

Если жилье найдено → можно бронировать

Если жилье не найдено → изменить параметры поиска

Если даты заняты → выбрать другие даты

Если оплата прошла → бронь подтверждена

Если оплата не прошла → попробовать снова

🧩 Component Diagram (Диаграмма компонентов)
Что это: Из каких частей состоит вся система
Простыми словами: Показывает отдельные программы, которые работают вместе:

Веб-сайт (то, что видят пользователи)

Сервис поиска (ищет жилье по параметрам)

Сервис бронирований (обрабатывает заказы)

Сервис платежей (принимает оплату)

База данных (хранит всю информацию)
И как они общаются между собой

💾 Базы данных (простое объяснение)
Что это: Места, где система хранит разную информацию
Простыми словами:

База пользователей - кто зарегистрирован в системе

База жилья - что сдается в аренду

База бронирований - кто что забронировал

База платежей - информация об оплате

База удобств - что есть в каждом жилье

Каждая диаграмма показывает систему с разных сторон, чтобы было понятно, как всё устроено и работает вместе.
