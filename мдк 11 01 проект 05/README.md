## Проект: автоматическое тестирование форм аутентификации (МДК 11 01 – проект 05)

**Цель:** комплексная система автоматического тестирования форм аутентификации (регистрация, вход, восстановление пароля) с веб-интерфейсом для просмотра результатов.

> 📖 **Новый ПК?** Смотрите [QUICK_START.md](QUICK_START.md) для быстрого запуска или [INSTALL.md](INSTALL.md) для подробной инструкции.

### 1. Требования

- PHP 8.0+
- Composer
- MySQL (для тестовой БД), либо совместимый сервер

### 2. Установка

1. Перейдите в папку проекта:

```bash
cd "мдк 11 01 проект 05"
```

2. Установите зависимости:

```bash
composer install
```

3. Создайте тестовую базу данных (например, `auth_project_test`) на основе основной БД из проекта «мдк 11 01 проект 03».

4. Примените тестовую схему (минимальный пример):

```bash
mysql -u root -p auth_project_test < tests/database/schema.sql
```

5. При необходимости отредактируйте параметры подключения к БД в `phpunit.xml` (секция `<php><env ...>`).

### 3. Структура проекта

- `src/`
  - `Validation/AuthValidator.php` – валидация телефонов и паролей.
  - `Services/AuthService.php` – упрощённый сервис регистрации, входа и восстановления пароля.
- `tests/`
  - `bootstrap.php` – базовый класс `BaseTestCase` (тайм-аут 1 секунда, изоляция транзакциями).
  - `Validation/ValidationTest.php` – **10 тестов валидации** (телефон, пароль, подтверждение пароля).
  - `Registration/RegistrationTest.php` – **15 тестов регистрации** (успех, ошибки, уникальность телефона, границы).
  - `Login/LoginTest.php` – **10 тестов входа** (успех, неверный пароль, отсутствующий пользователь, регистр).
  - `PasswordRecovery/PasswordRecoveryTest.php` – **15 тестов восстановления пароля** (коды, срок действия, сброс).
  - `database/schema.sql` – минимальная схема тестовой БД.
  - `logs/` – логи выполнения тестов (JUnit, вывод PHPUnit).
- `phpunit.xml` – конфигурация PHPUnit.
- `composer.json` – зависимости (PHPUnit, автозагрузка).
- `test_results.php` – веб-интерфейс для запуска тестов и просмотра результатов.

### 4. Запуск тестов

#### Через консоль

```bash
composer test
```

или

```bash
vendor/bin/phpunit --configuration phpunit.xml
```

### 5. Веб-интерфейс `test_results.php`

- Откройте в браузере файл `test_results.php` (например, через локальный сервер `http://localhost/.../test_results.php`).
- На странице:
  - Кнопка **«Запустить тесты»**.
  - Цветовая индикация статуса:
    - зелёный – успешно,
    - красный – ошибки/падения,
    - жёлтый – пропущенные.
  - Сводная статистика:
    - всего тестов,
    - количество ошибок и падений,
    - количество пропущенных тестов,
    - общее время выполнения.
  - Таблица по каждому тесту: класс, имя теста, статус, время, сообщение.
  - Сырой вывод PHPUnit для диагностики.

Интерфейс полностью русскоязычный и визуально интегрируется с современным веб-стилем (можно дополнительно подключить стили из основного проекта).

### 6. Авто-пропуск медленных тестов

- В `tests/bootstrap.php` базовый класс `BaseTestCase`:
  - измеряет время начала выполнения теста в `setUp()`;
  - в `tearDown()` вычисляет время выполнения;
  - если тест выполнялся дольше, чем `TEST_MAX_DURATION` (по умолчанию 1 секунда, задаётся через переменную окружения в `phpunit.xml`), тест помечается как **пропущенный**.

### 7. Изоляция тестов и очистка данных

- Подключение к БД выполняется один раз на процесс (PDO).
- Для каждого теста:
  - в `setUp()` запускается транзакция;
  - в `tearDown()` выполняется `ROLLBACK`, поэтому данные не сохраняются.
- Таким образом, каждый тест изолирован от других.

### 8. Интеграция с проектом «мдк 11 01 проект 03»

1. Скопируйте код системы аутентификации из проекта 03 в папку `src/` (или подключите через автозагрузку Composer).
2. При необходимости измените:
   - пространство имён и классы в тестах (`use App\Services\AuthService;` и т.д.);
   - структуру таблиц в `tests/database/schema.sql` под вашу реальную схему.
3. Убедитесь, что тестовая БД (`auth_project_test`) полностью повторяет структуру основной (без боевых данных).

### 9. Git и размещение в репозитории

1. Инициализируйте Git-репозиторий:

```bash
git init
git add .
git commit -m "auth form testing"
```

2. Убедитесь, что файл `.gitignore` исключает:
   - `vendor/`,
   - `tests/logs/`,
   - файлы `.env*`,
   - временные файлы.

3. Создайте приватный репозиторий на GitHub и добавьте его как удалённый:

```bash
git remote add origin git@github.com:ВАШ_ЛОГИН/мдк-11-01-проект-05.git
git push -u origin main
```

### 10. Дополнительно

- При необходимости вы можете:
  - Добавить новые тесты, расширяющие покрытие (например, дополнительные проверки валидации).
  - Подключить стили из основного проекта в `test_results.php`, чтобы интерфейс выглядел единообразно.
  - Тонко настроить порог времени `TEST_MAX_DURATION` в `phpunit.xml`.
