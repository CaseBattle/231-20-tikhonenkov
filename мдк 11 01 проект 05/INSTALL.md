# Инструкция по установке и запуску на новом ПК

## Шаг 1: Установка необходимого ПО

### 1.1. Установите PHP 8.0 или выше
- Скачайте PHP с официального сайта: https://www.php.net/downloads.php
- Или используйте XAMPP/OpenServer (включает PHP, MySQL, Apache)
- Проверьте установку в командной строке:
  ```bash
  php -v
  ``bash
  php -v`
  Должна быть версия 8.0 или выше.

### 1.2. Установите Composer
- Скачайте установщик: https://getcomposer.org/download/
- Или установите через XAMPP/OpenServer
- Проверьте установку:
  ```bash
  composer --version
  ```

### 1.3. Установите MySQL (если ещё не установлен)
- Используйте XAMPP/OpenServer (включает MySQL)
- Или установите отдельно: https://dev.mysql.com/downloads/mysql/
- Запустите MySQL сервис

## Шаг 2: Клонирование/копирование проекта

### Вариант А: Если проект в Git-репозитории
```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd "мдк 11 01 проект 05"
```

### Вариант Б: Если проект в архиве/папке
1. Распакуйте архив или скопируйте папку проекта
2. Откройте командную строку в папке проекта:
   ```bash
   cd "мдк 11 01 проект 05"
   ```

## Шаг 3: Установка зависимостей

В командной строке выполните:
```bash
composer install
```

Это установит PHPUnit и другие зависимости в папку `vendor/`.

## Шаг 4: Настройка базы данных

### 4.1. Создайте тестовую базу данных

Откройте MySQL (через phpMyAdmin, MySQL Workbench или командную строку):

**Через командную строку MySQL:**
```bash
mysql -u root -p
```

Затем выполните:
```sql
CREATE DATABASE auth_project_test CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

**Или через phpMyAdmin:**
- Откройте http://localhost/phpmyadmin
- Создайте новую базу данных `auth_project_test` с кодировкой `utf8mb4_unicode_ci`

### 4.2. Импортируйте схему базы данных

**Через командную строку:**
```bash
mysql -u root -p auth_project_test < tests/database/schema.sql
```

**Или через phpMyAdmin:**
- Выберите базу `auth_project_test`
- Перейдите на вкладку "Импорт"
- Выберите файл `tests/database/schema.sql` и нажмите "Вперёд"

### 4.3. Настройте параметры подключения (если нужно)

Откройте файл `phpunit.xml` и при необходимости измените параметры подключения к БД:

```xml
<env name="DB_HOST" value="127.0.0.1"/>      <!-- Хост БД -->
<env name="DB_PORT" value="3306"/>            <!-- Порт БД -->
<env name="DB_NAME" value="auth_project_test"/> <!-- Имя БД -->
<env name="DB_USER" value="root"/>            <!-- Пользователь БД -->
<env name="DB_PASSWORD" value=""/>           <!-- Пароль БД -->
```

**Если у вас другой пароль MySQL:**
- Измените `<env name="DB_PASSWORD" value="ваш_пароль"/>`

## Шаг 5: Проверка установки

### 5.1. Запуск тестов через командную строку

```bash
composer test
```

Или напрямую:
```bash
vendor\bin\phpunit --configuration phpunit.xml
```

**Ожидаемый результат:**
- Должны выполниться 50 тестов
- Все тесты должны пройти успешно (зелёный вывод)
- Если есть ошибки подключения к БД — проверьте шаг 4

### 5.2. Запуск через веб-интерфейс

1. **Запустите локальный веб-сервер:**

   **Вариант А: Встроенный сервер PHP (рекомендуется)**
   ```bash
   php -S localhost:8000
   ```

   **Вариант Б: XAMPP/OpenServer**
   - Скопируйте папку проекта в `htdocs` (XAMPP) или `domains` (OpenServer)
   - Запустите Apache через панель управления

2. **Откройте в браузере:**
   ```
   http://localhost:8000/test_results.php
   ```
   или
   ```
   http://localhost/мдк 11 01 проект 05/test_results.php
   ```

3. **Нажмите кнопку "Запустить тесты"**

4. **Проверьте результаты:**
   - Должна отобразиться статистика выполнения тестов
   - Все 50 тестов должны быть успешно пройдены

## Шаг 6: Устранение возможных проблем

### Проблема: "composer: команда не найдена"
**Решение:** 
- Убедитесь, что Composer установлен и добавлен в PATH
- Перезапустите командную строку

### Проблема: "PHPUnit не найден"
**Решение:**
```bash
composer install --no-dev
composer install
```

### Проблема: Ошибка подключения к БД
**Решение:**
1. Проверьте, что MySQL запущен
2. Проверьте параметры в `phpunit.xml` (хост, порт, пользователь, пароль)
3. Убедитесь, что база данных `auth_project_test` создана
4. Проверьте, что схема импортирована (должны быть таблицы `users` и `password_resets`)

### Проблема: "Класс не найден" или ошибки автозагрузки
**Решение:**
```bash
composer dump-autoload
```

### Проблема: Веб-интерфейс не запускает тесты
**Решение:**
1. Проверьте, что PHP может выполнять команды через `shell_exec()` или `exec()`
2. Проверьте права доступа к папке `tests/logs/` (должна быть возможность записи)
3. Проверьте путь к PHPUnit в `test_results.php` (для Windows: `vendor\bin\phpunit`, для Linux/Mac: `vendor/bin/phpunit`)

## Готово!

Проект должен быть полностью настроен и готов к работе. Вы можете:
- Запускать тесты через командную строку: `composer test`
- Просматривать результаты через веб-интерфейс: `test_results.php`
- Добавлять новые тесты в папку `tests/`
